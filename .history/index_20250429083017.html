Il est évident qu’avant de se lancer dans un tel projet, il est nécessaire de créer une maquette graphique en mobile first (Responsive design) Il faut également réfléchir à la structure de votre base de données. Pour cela, avant de commencer le développement, vous devez réaliser un modèle relationnel de données (MCD, MLD, MPD). Il vous faudra également installer un environnement Docker pour le projet avec toutes les dépendances nécessaires à votre travail de groupe. Voici une liste non exhaustive, mais avec les fonctionnalités minimales que votre site doit avoir : ➔ Page d’accueil attractive avec plusieurs sections dont une mise en avant des produits phares sur la page d’accueil / derniers produits mis en ligne. ➔ Design contemporain et respectant la charte graphique de votre entreprise ➔ Le site doit être responsive ! ➔ Barre de recherche de produits avec une autocomplétion en JavaScript asynchrone ➔ Accès à la boutique présentant tous les produits avec la possibilité de les filtrer par catégorie / sous-catégories sans rechargement de page ➔ Au clic sur chaque produit : une page « détails » complète et générée dynamiquement s’affiche (nom, image, prix, description, bouton d’ajout au panier...) ➔ Un système de création de comptes utilisateurs : ◆ Module inscription / connexion (Utilisation de JavaScript asynchrone obligatoire dans cette partie) _2 ◆ page de gestion du profil utilisateur (Récapitulatif et possibilité de modifier ses informations, voir son historique d’achat, consultation du panier...) ➔ Espace tableau de bord Administrateur : ◆ Gestion des produits à l’aide de back-office (Ajout, suppression, modification des produits, stocks...) ◆ Gestion des catégories et des sous-catégories de produits (Ajout, suppression, modifications...) ➔ Système de validation du panier (Vous n’êtes pas obligé d’implémenter une vraie solution de paiement, une simulation du processus suffit) Compétences visées ➔ Maquettage de projet/application ➔ Conception de base de données MCD, MLD, MPD ➔ Programmation Orientée Objet (POO) : utilisations des Classes ➔ Détailler un parcours utilisateur sur une fonctionnalité « métier » de votre site (Exemple : action d’achat) ➔ Structurer un projet, et penser à son architecture ➔ Faire de l’asynchrone avec JavaScript ➔ PHP ➔ SQL
